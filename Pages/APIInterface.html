API Interface

A World of Warcraft add-on which offers an interface for an otherwise text-based API documentation.

Looks neat... but what is it?

API Interface is a small yet helpful add-on for developers. It helps search through Blizzard's official API documentation by offering an easy way to navigate interface. In addition, users can search through global variables to find functions that aren't documented yet, or strings for specific formatting and localized texts. The goal of this add-on is to help the user navigate through the documentation, and quickly find what they are looking for. As a result, it helps speed up development.

The itch that needed stratching

In patch 7.2, Blizzard started adding official documentation for their API. Players can search through this documentation through chat commands, which prints the results into the chat window. It's a step in the right direction over not having any documentation at all, but the navigation proved to be less than favourable. Other factors such as chat channels or guild chat could easily bump the results out of view. The default chat limits the mount of text it can contain, and its size limits how much is visible at once. For exmaple, if the user searches for 'player', it won't show relevant functions as they have been bumped out by the rest of the results.

Only half of the info fits in the default chatwindow size.

The thought process

The most important change was to get the information out of the chat window. That way it wouldn't be affected by other chat messages and its limits. The API is something only a very small amount of the userbase would ever come in contact with. This meant there wasn't really anything already existing to incoporate it into. The best solution was to just make a new window. I could have just copied the returned documentation one to one into the new window, but then it would still be hard to read. Instead, I took inspiration from the achievements window. In this window all achievements are listed with just their name, and the player can expand them if they want more information. As the documentation API returns its information in tables, I could easily turn every table entry into a separate entry in the list. As every piece of the documentation comes pre-formatted, it could be presented directly with its consistent color coding. Because of this, the add-on also requires minimal maintenance with future updates. Every new piece of documentation will automatically get its own entry in the list. As not all API is updated and documented yet, there are still some functions that won't show up in the list. To combat this, I added several buttons that allow the user to search for global variables and functions. Typing something in the search box and clicking one of the buttons will open a separate window with a list of all relevant results.

Some functions aren't part of the documentation yet, but can still be of use when developing.
